# CodeRabbit レビュー要約ログ案内

このファイルは入口とテンプレートだけを置く。実ログは月次ファイルへ分割する。

## 現在の記録先

- [2026-03 ログ](coderabbit-review-logs/2026-03.md)

## 使い方

1. まず [CodeRabbit Review Workflow](coderabbit-review.md) で運用ルールを確認する。
2. PR 完了後に、この案内から当月ログを開いて要約を追記する。
3. 月が変わったら `docs/process/coderabbit-review-logs/YYYY-MM.md` を追加し、このページの「現在の記録先」を更新する。

## 記録テンプレート

```markdown
## YYYY-MM-DD PR #<番号> <タイトル>
- レビュー状況: `COMMENTED` / `APPROVED` / `SUCCESS` など
- 要約:
  - （重要指摘の要点）
- 指摘一覧:
  - [採用] （指摘）
  - [非採用] （指摘と理由）
- この作業で当てはまりそうな内容（横展開候補）:
  - （同系統の壊れやすい箇所、再利用できる改善案）
- 実施:
  - （実際に行った修正）
- 残課題:
  - （次回に回した項目があれば）
```

## 分割ルール

- 1 ファイル = 1 か月を基本にする。
- 入口ファイルにはテンプレート、当月リンク、分割ルールだけを置く。
- 過去ログを参照するときは必要な月だけ開く。
