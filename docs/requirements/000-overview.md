# VBA for Excel Language Extension 要件定義（Draft）

## 0. 前提とゴール

### 0.1 前提（固定）

* 対象は **Excel VBA のみ**
* **Option Explicit を前提**（未宣言変数はエラーとして診断）
* **Win64 必須**

  * `PtrSafe` を必須扱い（Declare文などで検査）
  * `LongPtr` / `LongLong` など Win64 で重要な型を優先サポート
* “VBEの実行環境”や“マクロ実行”はスコープ外（編集支援に限定）

### 0.2 ゴール（ユーザー体験）

* VS Codeで `.bas/.cls/.frm` を開いて、以下が安定して使える

  * 構文チェック（壊れた状態でも診断が出る）
  * IntelliSense（補完/候補の絞り込み/引数ヒント）
  * 定義ジャンプ・参照検索
  * セマンティックハイライト（識別子の意味で色分け）
  * 型推論（Excel VBAで現実的に役立つ範囲）

---

## 1. 対象範囲

### 1.1 入力ファイル

* `.bas`（標準モジュール）
* `.cls`（クラスモジュール）
* `.frm`（フォーム）

  * 初期は **frmのフォーム定義部は“テキスト扱い”**、コード部のみ解析（段階対応）

### 1.2 VBA言語要素（優先度）

* 最優先：宣言・手続き境界・名前解決に必要な構文

  * `Option Explicit`
  * `Sub/Function/Property Get/Let/Set`
  * `Dim/Private/Public/Const/Enum/Type`
  * `Declare PtrSafe Function/Sub`（Win64）
  * 制御構文：`If/For/Do/Select/With/On Error`
* 低優先：VBAの隅・互換性要素

  * 条件付きコンパイル（`#If VBA7 Then` 等）は初期は限定/後回し可

---

## 2. 機能要件（MUST/SHOULD/COULD）

### 2.1 MUST（MVP成功条件）

#### A. ハイライト

* シンタックスハイライト（TextMate）

  * コメント / 文字列 / キーワード / 数値 / 識別子 / 予約語 / 属性行（`Attribute VB_Name = ...`）など

#### B. 解析（最低限）

* 字句解析（Lexer）

  * 行継続（` _`）、`'`コメント、`Rem`、`""`エスケープ、日付`#...#`、型サフィックス（`$ % & ! # @`）を扱う
* 構文解析（Parser）

  * 「モジュール」→「手続き」→「文」まで、壊れたコードでも継続できる（エラー回復）
* シンボル抽出（AST→Symbol Table）

  * 手続き名、引数、戻り値、修飾子（Public/Private/Friend/Static）
  * モジュールレベルの `Const/Enum/Type/Declare`、変数宣言

#### C. 診断（Diagnostics）

* 構文エラー（パース不能箇所）
* **未宣言変数**（Option Explicit前提）
* `Declare` の Win64/PtrSafe要件

  * `Declare` 文で `PtrSafe` が無い場合はエラー
  * ポインタ相当（ハンドル等）で `Long` を使っている疑い、等は最初は「警告」扱い（段階導入）

#### D. IntelliSense（最小）

* Completion：同一ファイル内（モジュール内）の

  * 変数/定数/Enum/関数/Sub/Property
* Go to Definition：同一ファイル内
* Document Symbols：アウトライン（モジュール・手続き一覧）

---

### 2.2 SHOULD（“実用域”）

* ワークスペース全体解析

  * 複数モジュール間の Public シンボル参照解決
* Find References（参照検索）
* Rename（安全な範囲：ローカル変数→段階的に拡張）
* Signature Help（引数ヒント）
* 基本の型推論（現実に効く範囲）

  * `As Type` 明示
  * リテラル（数値/文字列/日付/Boolean/Nothing）
  * 単純代入 `a = b`
  * 関数戻り型（`As Type`、戻り変数代入）
  * 配列（`()`、`ReDim`の限定的追跡）

---

### 2.3 COULD（“強い”）

* セマンティックハイライト（Semantic Tokens）

  * 変数/定数/引数/関数/型/列挙/プロパティ/クラス/モジュール など
* コードアクション

  * 未宣言変数の `Dim` 自動追加（適切なスコープに）
  * `PtrSafe` 追加の提案、`Long`→`LongPtr`候補など（安全な範囲から）
* Excelオブジェクトモデルの型スタブ

  * `Application/Workbook/Worksheet/Range` 等の基本型
  * `.Value/.Value2/.Cells/.Range` 等の主要メンバー補完（スタブベース）
* Withブロック内の `.` 解決（段階的）
* 既定メンバー/既定プロパティの限定対応（必要性が高い箇所のみ）

---

## 3. 非機能要件

### 3.1 性能

* タイピング中の解析はデバウンス（例：200–500ms）
* パースは差分更新 or ファイル単位再解析（初期は再解析でも可、将来差分へ）
* ワークスペース解析はバックグラウンドスレッド/別プロセス（Language Server）

### 3.2 正確性ポリシー

* 解析不能・推論不能は「Unknown」として扱い、**誤検知を減らす**
* 高度推論よりも **“壊れない/遅くならない”** を優先

### 3.3 保守性

* レイヤ分離（必須）

  * Lexer / Parser / AST
  * Symbol Table / Type System / Inference
  * Diagnostics Rules（ルール単位で独立）
  * LSPハンドラ（UI層）
* テスト

  * パース木スナップショット（Golden）
  * 診断（入力→期待するエラー一覧）
  * 補完（位置→候補セット）

## 4. 解析仕様（VBA特有の取り扱いを明文化）

### 4.1 字句仕様（最低限確定）

* 行継続：行末 ` _`（空白・コメント混在を含む）
* コメント：`'` と `Rem`
* 文字列：`""` を `"` として扱う
* 型サフィックス：`$ % & ! # @`
* 日付リテラル：`#...#` をトークン化（解釈は後回し可）
* 属性行：`Attribute VB_...` を専用トークン扱い（解析の邪魔をしない）

### 4.2 名前解決（最初のルール）

* スコープ優先順位（案）

  1. 手続きローカル
  2. モジュールレベル
  3. 同一プロジェクト内 Public
  4. 型スタブ（Excel/VBA標準）
* Option Explicit前提：未宣言はエラー（ただし外部型スタブが拾える場合は除外）

### 4.3 Win64/PtrSafe ルール（MUST）

* `Declare` 文には `PtrSafe` 必須
* `LongPtr` を理解（型として存在）
* `PtrSafe` なしDeclareは **エラー**
* 型の危険検出（Longでポインタ相当など）は **警告**から開始（誤検知対策）

---

## 5. 実装アーキテクチャ案（推奨）

### 5.1 VS Code側

* TextMate Grammar：シンタックスハイライト
* Language Server（LSP）：診断/補完/ジャンプ/セマンティックトークン等

### 5.2 解析方式

* 手書き再帰下降（VBA特化）→ AST → 型推論
* MVPでは「手続き境界＋宣言」を最優先に、文の詳細は段階導入

---

## 6. マイルストーン

* M0：拡張雛形＋TextMateハイライト
* M1：トップレベル解析（宣言/手続き境界）＋Diagnostics（構文・未宣言・PtrSafe）
* M2：LSP最小（補完・定義ジャンプ・アウトライン）
* M3：ワークスペース解析＋参照検索＋リネーム（限定）
* M4：型推論拡張＋セマンティックハイライト＋コードアクション

---

## 7. “やらないこと”（初期）

* VBA実行・デバッグ機能
* 参照設定（VBEの参照ライブラリ）を完全再現する精密型解決
* 既定プロパティ/既定メンバーの完全再現（必要箇所のみ段階対応）
* 条件付きコンパイルの完全対応
