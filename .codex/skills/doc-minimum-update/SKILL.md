# Doc Minimum Update

## skill 名
`doc-minimum-update`

## 目的
実装差分に直接関係する文書だけを、必要最小限で更新する。主成果物は常にコード変更とする。

## いつ使うか
- 実装後、または同一タスク内で実装完了が確実なとき
- 更新しないと説明不足、使用方法の不整合、手順不整合、レビュー困難のいずれかが出るとき
- README、コメント、使用例、手順差分の最小修正で足りるとき

## 使ってはいけないケース
- 実装前
- 抽象的な指針整理や将来のためだけの整理をしたいだけのとき
- 広範囲な再編、章立て変更、まとめ直しが必要なとき
- 判断に迷うとき

## 入力として確認すべきこと
- 実装差分の内容
- 変更されたコマンド、引数、振る舞い、制約
- どの文書がその差分を直接説明しているか
- 更新しない場合に何が不整合になるか

## 実行手順
1. 実装差分を確認する。
2. どの文書が実装差分に直接関係するかを特定する。
3. 更新しないと不整合が出る箇所だけを抜き出す。
4. 最小差分で更新する。
5. 更新理由を短く説明できる状態にする。

## 出力ルール
- 実装差分に直接関係する文書だけを更新する
- README、コメント、使用例、手順差分の最小修正だけを許可する
- 更新理由は 1〜3 行で説明できる状態にする
- 文書のみでタスクを完了してはならない

## 禁止事項
- 抽象的な指針整理
- 将来のためだけの整理
- 広範囲な再編
- 実装前の先回り更新
- 変更量が最小限でない更新
- 迷ったままの更新

迷ったら更新しない。迷ったら実装優先。

## 例
- README の該当コマンドだけ修正する
- 新しい引数が増えた関数の説明を 1 段落だけ追記する
- 変更のない章立てや設計思想には触れない
